## Phase 3: Notifications & Metrics

<Badge text="Planned" variant="default" />

Stay informed about build status without checking the dashboard.

<CardGrid>
  <Card title="Slack Integration" icon="discord">
    Connect your Slack workspace. Get notified when builds start, succeed, or fail. Rich messages with commit info and build links.
  </Card>
  <Card title="Email Notifications" icon="email">
    Email alerts on build failures. Optional daily digest of build activity across all repositories.
  </Card>
  <Card title="Webhook Notifications" icon="external">
    POST to any URL when build events occur. Customizable JSON payloads for integration with any system.
  </Card>
  <Card title="Build Metrics" icon="graph">
    Track build duration trends, success rates, and queue times. Identify slow builds and flaky tests.
  </Card>
</CardGrid>

## Phase 4: Team Features

<Badge text="Future" variant="caution" />

Multi-user support for teams sharing a build server.

<CardGrid>
  <Card title="User Accounts" icon="seti:lock">
    Email/password authentication. Invite team members. Session management.
  </Card>
  <Card title="Role-Based Access" icon="seti:folder">
    Roles: Owner, Admin, Developer, Viewer. Control who can trigger builds vs. who can only watch.
  </Card>
  <Card title="Organizations" icon="group">
    Group repositories by team or project. Organization-level settings and permissions.

## Non-Goals

To keep scope focused, we won't build:

- **Hosted version** ‚Äî Oore is self-hosted only. No SaaS.
- **Device testing** ‚Äî Use Firebase Test Lab for simulators/emulators.
- **Windows/Linux agents** ‚Äî macOS only (that's where Xcode lives).
- **Custom scripting language** ‚Äî Shell + YAML is enough.
