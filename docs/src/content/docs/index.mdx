## How It's Different

With hosted CI, you upload signing credentials to their cloud and run builds on their machines.

**With Oore**, your Mac mini or Mac Studio becomes the CI server. Credentials never leave Keychain. You connect to it remotely.

```
┌─────────────────────┐                    ┌─────────────────────────────────────┐
│   Your Laptop       │                    │         Your Mac (the server)        │
│                     │                    │                                      │
│  ┌───────────────┐  │      HTTPS         │  ┌──────────┐    ┌───────────────┐  │
│  │  oore (CLI)   │──┼───────────────────▶│  │  oored   │───▶│    Keychain    │  │
│  └───────────────┘  │                    │  │ (server) │    │  certs/profiles│  │
│                     │                    │  └──────────┘    └───────────────┘  │
│  ┌───────────────┐  │      HTTPS         │       │                             │
│  │    Browser    │──┼───────────────────▶│       ▼                             │
│  └───────────────┘  │                    │  ┌──────────┐    ┌───────────────┐  │
│                     │                    │  │  SQLite  │    │   Artifacts    │  │
└─────────────────────┘                    │  │    DB    │    │   .ipa/.apk    │  │
                                           │  └──────────┘    └───────────────┘  │
         GitHub/GitLab ────webhooks───────▶│                                      │
                                           └─────────────────────────────────────┘
```

## Hosted CI vs Oore

| | Hosted CI | Oore |
|---|-----------|------|
| **Credentials** | Upload to their cloud | Stay in your Keychain |
| **Code** | Runs on their VMs | Runs on your Mac |
| **Cost** | Per build minute | Fixed hardware cost |
| **Queue** | Shared | Dedicated |

## Current Status

**Early development.** The core feature (build execution) doesn't exist yet.

What works: webhook ingestion, repository management, service management, REST API, CLI with multi-server profiles, web UI shell.

What doesn't: builds, code signing, artifacts, publishing.
