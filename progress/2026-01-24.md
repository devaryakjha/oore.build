# Progress: 2026-01-24

## Summary

Initial project scaffolding for Oore - a self-hosted Codemagic alternative for Flutter CI/CD.

## What We Built

### Rust Workspace
- **oore-core**: Shared library for types, database, and business logic
- **oore-server** (`oored`): Axum HTTP server on port 8080
  - `GET /api/health` - returns `{"status":"ok"}`
  - `GET /api/version` - returns version info
- **oore-cli** (`oore`): CLI client with clap
  - `oore health` - checks server connectivity
  - `oore version` - shows CLI and server versions

### Next.js Frontend
- Created with shadcn/ui using bun
- Preset: Lyra style, amber theme, JetBrains Mono font, hugeicons

## Key Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Package manager | **bun** | Faster than npm, modern |
| UI framework | **shadcn/ui** | Composable, not a dependency |
| Icon library | **hugeicons** | Part of shadcn preset |
| Server binary name | **oored** | Unix daemon convention (d suffix) |
| CLI binary name | **oore** | Clean, matches project name |
| Database | **SQLite** (planned) | Embedded, no external deps |

## Tech Stack (Confirmed)

- **Backend**: Rust (Axum 0.8, SQLx 0.8, Tokio)
- **Frontend**: Next.js 16, React 19, Tailwind v4, shadcn/ui
- **Database**: SQLite (embedded)
- **Package manager**: bun (strictly)
